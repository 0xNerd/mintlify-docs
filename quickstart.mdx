## 1. Prerequisites

- Node.js 20+
- Supabase project + service role key
- Privy app ID + secret
- Privacy Cash SDK access (or the bundled submodule)
- Telegram bot token from [@BotFather](https://t.me/botfather)
- Solana RPC endpoint (mainnet or devnet)

<Tip>
  Use separate Supabase + Privy projects per environment. The bot reads the
  `ENVIRONMENT` variable to decide which credentials to load.
</Tip>

## 2. Install dependencies

```bash
git clone <your-fork-of-zk-send>
cd zk-send
npm install
```

If you modified the Privacy Cash SDK locally, rebuild it before launching the bot:

```bash
npm run build:sdk
```

## 3. Configure environment variables

```bash
cp env.example .env
```

Update `.env` with:

- `BOT_TOKEN`, `BOT_NAME`
- `PRIVY_APP_ID`, `PRIVY_APP_SECRET`
- `SUPABASE_URL`, `SUPABASE_SERVICE_KEY`
- `SOLANA_RPC_URL`
- `PRIVACY_CASH_API_KEY`

<Card icon="circle-info" title="Need a full list?">
  See `src/config/env.ts` for descriptions and defaults that are applied per environment.
</Card>

## 4. Seed Supabase

Apply the schema so the bot can store users, wallets, and transaction logs:

```bash
psql < supabase_url > -f src/database/schema.sql
```

Or use the Supabase SQL editor and paste the same file contents.

<Note>
  Enable Row Level Security (RLS) on all tables before exposing the Supabase
  project to production traffic.
</Note>

## 5. Run the bot

```bash
# Watch mode for local development
npm run dev

# Production build
npm run build && npm start
```

The Telegraf webhook automatically registers itself if `WEBHOOK_URL` is set. If not, the bot falls back to long polling.

## 6. Send SOL privately

1. Open Telegram and DM your bot.
2. Run `/start` to create or restore your Privy account. First-time users can keep the auto-generated embedded wallet or import an existing key inline.
3. Use `/wallets` later to switch or add additional wallets.
4. Use `/send` to pick an amount, recipient, and confirm the Privacy Cash transfer.
